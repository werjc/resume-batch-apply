### 一句话概述
独立设计并开发了一款 Chrome 浏览器扩展，支持 BOSS直聘、智联招聘等 8 个主流招聘网站的一键批量投递，集成 AI 大模型进行岗位风险分析，解决求职过程中效率低和诈骗风险高的问题。

### 项目描述（简历用）

- **产品设计**：深入分析求职者多平台投递痛点，发现最大瓶颈在于重复性点击和诈骗岗位识别。设计批量投递、深度爬取、AI 风险分析三大核心功能，形成完整的求职效率与安全工具。
- **技术实现**：独立完成 Chrome 扩展的架构设计、编码和 20+ 轮迭代。采用适配器模式支持 8 个不同招聘网站，每个网站独立解析和交互逻辑。使用 SPA 路由拦截技术实现对现代单页应用的无刷新页面切换感知。
- **AI 集成**：集成 LLM API（DeepSeek/OpenAI），实现岗位风险自动分析（诈骗识别）和公司类型智能判断。支持用户自定义 API 提供商、模型和 endpoint。
- **工程质量**：通过 20+ 轮版本迭代，修复 DOM 选择器匹配、SPA 状态同步、跨域请求、竞态条件等核心工程问题。项目具备完整的模块化结构（22 个源文件、~4,500 行代码）。

## 面试问答准备

### Q: 这个项目主要是做什么的？
**A**: 它是一个 Chrome 浏览器扩展，帮求职者在一个面板里管理多个招聘网站的投递。打开 BOSS直聘等招聘网站的搜索结果页，扩展自动识别当前网站、解析所有岗位。你可以筛选条件、勾选想要投递的岗位，点击"立即投递"自动完成投递。还有 AI 分析模块，接入大模型自动判断岗位是否有诈骗风险。

### Q: 你自己不懂编程，这个项目是怎么做出来的？
**A**: 我使用 AI 辅助开发工具（Claude Code）。我的角色是产品经理兼测试者——定义需求、设计交互、发现 bug、反馈迭代。整个过程中我学会了：需求拆解、bug 复现与描述、用非技术语言沟通技术问题、软件版本管理。这让我理解了软件开发的完整生命周期。

### Q: 技术上有哪些难点？怎么解决的？
**A**:
1. **8 个网站 DOM 结构完全不同**：用适配器模式，每个网站独立模块，共享基类。基类用 `getComputedStyle` 分析视觉属性而非硬编码 class 名来提取标题。
2. **SPA 路由不刷新页面**：五重监听（pushState/replaceState/popstate/hashchange/800ms 轮询）。
3. **公司类型在搜索结果页看不到**：先用 300+ 关键词排除法（默认民营），再提供可选的 LLM API 接入做智能判断。
4. **深度爬取卡死**：改用 URL 拼接优先 + 20 秒硬超时 + stopSignal 即时停止。
5. **AI API 调用失败**：排查发现 `host_permissions` 未覆盖 API 域名，修复后加入详细日志链路。

### Q: 从项目中学到什么？
**A**: 
1. 完整的软件产品开发：需求分析 → 架构设计 → 编码 → 测试 → 20+ 轮迭代优化
2. Chrome 扩展工作原理：Content Script / Service Worker / 消息通信
3. 设计模式实践：适配器模式让 8 个网站共享核心逻辑
4. AI 辅助编程方法论：有效 prompt 编写、bug 描述、code review
5. 用户体验设计：侧边栏 vs 弹窗、粘性布局、防呆设计

### Q: 如果重新做，怎么改进？
**A**:
1. 先深度调研 2-3 个核心网站，打磨到完美再扩展
2. 用 TypeScript 重写减少类型错误
3. 加入自动化测试（每次改动自动验证 8 个网站解析）
4. 加入使用数据埋点指导迭代方向
