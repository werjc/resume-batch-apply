### 一句话概述
独立设计并开发了一款 Chrome 浏览器扩展，支持 BOSS直聘、智联招聘等 8 个主流招聘网站的一键批量投递，集成 AI 大模型进行岗位风险分析、简历匹配评分，解决求职过程中效率低和诈骗风险高的问题。

### 项目描述（简历用）

- **产品设计**：深入分析求职者多平台投递痛点，发现最大瓶颈在于重复性点击和诈骗岗位识别。设计批量投递、AI 风险分析、简历匹配评分三大核心功能，形成完整的求职效率与安全工具。
- **技术实现**：独立完成 Chrome 扩展的架构设计、编码和 30+ 轮迭代。采用适配器模式支持 8 个不同招聘网站，每个网站独立解析和交互逻辑。v3.0 用 MutationObserver 瀑布流替代传统翻页爬取方案。
- **AI 集成**：集成 LLM API（DeepSeek/OpenAI），实现岗位风险自动分析（诈骗识别）、公司类型智能判断、简历匹配度评分（0-100）。支持 PDF 简历上传自动提取文本。
- **工程质量**：通过 30+ 轮版本迭代，发现并修复了 Chrome 扩展消息传递陷阱（content script 自投递问题）、DOM 解析竞态条件、AI 分析状态泄漏等核心工程问题。项目具备完整的模块化结构。
- **UI 设计**：v3.1 完成全界面视觉重设计——现代玻璃态卡片风格，CSS 变量系统、渐变 header、光泽动画、弹性过渡。无障碍与交互细节打磨。

## 面试问答准备

### Q: 这个项目主要是做什么的？
**A**: 它是一个 Chrome 浏览器扩展，帮求职者在一个面板里管理多个招聘网站的投递。打开 BOSS直聘等招聘网站的搜索结果页，扩展自动识别当前网站、解析所有岗位、实时监控页面滚动加载的新岗位。你可以筛选条件、勾选目标岗位，点击"立即投递"自动完成。AI 分析模块不仅判断诈骗风险，还能根据你的简历给每个岗位打匹配分。

### Q: 你自己不懂编程，这个项目是怎么做出来的？
**A**: 我使用 AI 辅助开发工具（Claude Code）。我的角色是产品经理兼测试者——定义需求、设计交互、发现 bug、反馈迭代。整个过程中我学会了：需求拆解、bug 复现与描述、用非技术语言沟通技术问题、软件版本管理、Git 工作流、MCP 服务器配置、Claude Code Skill 安装。这让我理解了软件开发的完整生命周期。

### Q: 技术上有哪些难点？怎么解决的？
**A**:
1. **8 个网站 DOM 结构完全不同**：适配器模式，基类用 `getComputedStyle` 分析视觉属性而非硬编码 class 名。
2. **SPA 路由不刷新页面**：五重监听（pushState/replaceState/popstate/hashchange/面板开启轮询），面板关闭停止轮询节省 CPU。
3. **深度爬取卡死**（v2.x 版本遗留问题）：发现根因是 Chrome 扩展消息传递机制——content script 发出的 `sendMessage` 不会回传给自身，导致 UI 冻结永不恢复。v3.0 彻底删除爬取方案，改用 MutationObserver 瀑布流。
4. **PDF 简历文本提取失败**：初版用 `String.fromCharCode.apply` 对大文件抛 RangeError；改用 `DecompressionStream('deflate-raw')` 解压 FlateDecode 流后成功提取。
5. **公司类型和学历筛选无效**：默认值 `'private'` 和 `'bachelor'` 导致 95% 岗位归为同类。改为 `'unknown'` 和 `'none'`，配合 AI 分析自动填充。
6. **AI 分析超过 10 条后卡死**：全部岗位预标记为 analyzing，但只更新前 10 条。改为只标记前 10 条。

**A**: 
1. 完整的软件产品开发：需求分析 → 架构设计 → 编码 → 30+ 轮迭代优化
2. Chrome 扩展工作原理：Content Script / Service Worker / Popup / 消息通信
3. 设计模式实践：适配器模式 + 观察者模式（MutationObserver）

5. UI/UX 设计：侧边栏 vs 弹窗、瀑布流 vs 翻页、玻璃态视觉设计

**A**:

2. 用 TypeScript 重写减少类型错误
3. 加入自动化测试（每个网站解析验证）
4. 加入使用数据埋点指导迭代方向
5. 浏览器原生 Side Panel API 替代注入式面板
