### 一句话概述
> 独立设计并开发了一款 Chrome 浏览器扩展，支持在 BOSS直聘、智联招聘等 8 个主流招聘网站上一键批量投递简历，解决求职过程中重复操作效率低的问题。

### 项目描述（简历用）

- **产品层面**：深入分析了求职者在多平台投递简历的痛点，发现最大瓶颈在于重复性点击操作，而非简历内容准备。据此确定了"批量投递"作为核心功能，同时设计了筛选、定时、爬取等辅助功能，形成完整的求职效率工具。
- **技术层面**：独立完成了 Chrome 扩展的架构设计、编码实现和测试迭代。采用适配器模式支持 8 个不同招聘网站，每个网站有独立的页面解析和交互逻辑。使用 SPA 路由拦截技术实现了对现代单页应用的无刷新页面切换感知。
- **工程层面**：通过 16 个版本的迭代，修复了 DOM 选择器匹配、SPA 状态同步、跨域数据抓取等核心问题，将岗位识别率从 30% 提升到 95%+。项目具备完整的模块化结构和 19 个源文件。

## 面试问答准备

### Q: 这个项目主要是做什么的？
**A**: 它是一个 Chrome 浏览器扩展，帮助求职者在一个面板里管理多个招聘网站的投递。你打开 BOSS直聘 或者 智联招聘 的搜索结果页，扩展会自动识别当前网站，解析出所有岗位，你可以勾选想要投递的岗位，点击"立即投递"就能自动完成投递操作，不需要一个一个手动点击。

### Q: 你自己不懂编程，这个项目是怎么做出来的？
**A**: 我使用了 AI 辅助开发工具（Claude Code）。我的角色是产品经理和测试者——定义需求、设计交互流程、发现 bug、反馈问题。AI 负责代码生成。整个过程我学会了如何与 AI 协作完成软件开发：把需求拆解成可执行的任务、描述bug时附上复现步骤和截图、用非技术语言表达技术问题。这个项目让我理解了软件开发的完整生命周期。

### Q: 技术上有哪些难点？你是怎么解决的？
**A**: 
1. **8个网站结构完全不同**：用适配器模式，每个网站写独立的解析模块，共享同一个基类
2. **BOSS直聘是单页应用，切换tab不刷新**：用拦截 `history.pushState` + URL 轮询来感知页面变化
3. **岗位卡片每个网站用的 class 名都不一样**：改成了分析 DOM 的视觉属性（字号、字重、位置）来找标题，不依赖 class 名
4. **公司类型在搜索结果页不显示**：加了联网查询功能，自动 fetch 公司详情页来提取公司类型

### Q: 你从这个项目中学到了什么？
**A**: 
1. 完整的软件产品开发流程：需求分析 → 架构设计 → 编码 → 测试 → 迭代优化
2. 浏览器扩展的工作原理：Content Script、Service Worker、消息通信
3. 设计模式的价值：适配器模式让 8 个网站共享同一套核心逻辑
4. AI 辅助编程的方法：如何写有效的 prompt，如何描述 bug，如何做 code review
5. 用户体验设计：侧边栏 vs 弹窗的取舍、拖拽 vs 固定的选择

### Q: 如果让你重新做这个项目，你会怎么改进？
**A**:
1. 先做用户调研，确定最核心的 2-3 个网站，把投递流程打磨到完美再扩展
2. 用 TypeScript 重写，减少运行时类型错误
3. 加入自动化测试，每次改代码后自动验证 8 个网站是否都能正常解析
4. 加入使用数据埋点，了解用户最常用哪些功能，指导迭代方向
